## TL;DR

OpenClaw 不是把聊天能力包成 bot，而是以 SQLite control DB 為真相來源、以 control plane 作判斷中心、再把 public site 當成 projection 的單機 deterministic system。

## Why it matters

呢頁幫人分清 OpenClaw 的真相來源同 public projection 邊界，避免把網站、Slack thread 或單次對話誤當成可回放的 control truth。

## Core model / main idea

- SQLite control DB 先係 live task truth；memory、Slack thread 同 public site 都只係輔助表面。
- Gateway 只做 ingress，control plane 負責 routing、policy、verification、review，同 public status 決策。
- worker runner 只租約執行 queued attempts，唔可以直接宣告 public 完成。
- Phase 15 將 canonical bundle 同 public projection 分開，令 projection 可以重建、回放同撤回。

## Examples / failure modes / usage

- 使用：先對住 runbook 睇 system boundary，再用 Phase 15 contract 理解 bundle、review queue、publish queue 點樣串起。
- 失效模式：如果將 public site 當真相來源，receipt 同 review gate 就會同 control DB 脫鈎。
- 失效模式：如果 worker 直接對外宣告完成，就會跳過 control plane 同 review gate。
- 使用：當 public 內容需要刷新時，應由 private canonical output 重投影，而唔係由 public repo 反向修正 truth。
