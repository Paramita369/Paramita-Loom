## 先看結論

如果你想盡快在 Mac 上開始用 OpenClaw，照住下面 5 步做就可以：安裝 CLI、跑首次設定、檢查 gateway、打開 dashboard、最後跑一次自檢。

`onboarding` 即是首次設定精靈；`gateway` 即是 OpenClaw 在你電腦上的入口服務；`dashboard` 即是本機管理畫面。

<div class="loom-note">
  <strong>最短成功路徑</strong>
  <p>裝好 `openclaw` CLI，跑一次首次設定精靈（onboarding），確認 gateway 有回應，再打開 dashboard，最後用 `openclaw doctor` 做一次自檢。</p>

## 步驟

### <span id="step-1">1. 安裝 CLI</span>

先安裝 `openclaw` 指令。CLI 即是你之後會在 Terminal 輸入的主要命令。

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
exec $SHELL -l
openclaw --version
```

做完之後，你應該見到 `openclaw --version` 印出版本號，而唔係 `command not found`。

[遇到 `command not found`？看錯誤 1](#error-1)

### <span id="step-2">2. 跑首次設定</span>

首次設定精靈會帶你做第一次設定，並把背景常駐程式一併裝好。

```bash
openclaw onboard --install-daemon
```

根據 [官方 Onboarding 文件](https://docs.openclaw.ai/start/wizard)，精靈會一步步帶你設定 gateway、workspace、登入方式，同埋第一輪健康檢查。今篇建議先用 QuickStart 完成第一次設定。

做完之後，你至少應該回到可繼續輸入命令的 Terminal，並且唔會停喺未完成設定的中途畫面。

[首次設定一開始就要求你處理舊設定？看錯誤 2](#error-2)

### <span id="step-3">3. 檢查 gateway</span>

第一次設定完成後，先確認 gateway 已經起好。你可以把它理解成 OpenClaw 在你電腦上的入口服務。

```bash
openclaw gateway status
```

做完之後，你應該見到狀態摘要，而唔係連線失敗或找不到 gateway。

[gateway 起唔到？看錯誤 3](#error-3)

### <span id="step-4">4. 打開 dashboard</span>

dashboard 即是本機管理畫面。當 gateway 正常之後，再打開 dashboard。

```bash
openclaw dashboard
```

做完之後，你應該見到本機畫面打開，或者命令列至少印出可打開的本機網址。

[dashboard 冇打開？看錯誤 4](#error-4)

### <span id="step-5">5. 跑自檢</span>

最後跑一次自檢，確認目前冇阻擋你繼續用落去的致命問題。

```bash
openclaw doctor
```

做完之後，你應該見到檢查結果；如果有錯，重點睇係咪出現阻擋你繼續使用的致命錯誤。

[自檢出現阻擋性錯誤？看錯誤 5](#error-5)

## 驗證方式

你完成上面 5 步之後，只要下面 5 件事都成立，就代表主線安裝大致完成：

- `openclaw --version` 有正常回傳版本號。
- 首次設定精靈可以完整跑完。
- `openclaw gateway status` 有回到狀態摘要。
- `openclaw dashboard` 有打開本機畫面，或者印出可打開的網址。
- `openclaw doctor` 沒有出現阻擋你繼續使用的致命錯誤。

如果其中一項未過，直接跳去下面的 `常見錯誤`，再按對應連結回到相關步驟。

## 常見錯誤

### <span id="error-1">錯誤 1：`openclaw: command not found`</span>

通常係安裝後仲未重開 shell。先執行 `exec $SHELL -l`，再試 `openclaw --version`；如果仍然唔得，再重跑安裝步驟。

[返回步驟 1](#step-1)

### <span id="error-2">錯誤 2：首次設定精靈要求你處理舊設定</span>

這通常代表你機器上已經有 `~/.openclaw/openclaw.json`。官方流程容許你保留、修改或重設；如果它提示 legacy config 問題，先跑 `openclaw doctor`，再決定要保留還是重設。

[返回步驟 2](#step-2)

### <span id="error-3">錯誤 3：`openclaw gateway status` 顯示 gateway 未起好</span>

先重新跑 `openclaw onboard --install-daemon`，確認你在首次設定精靈裡選的是 local gateway 路線，之後再重試狀態檢查。

[返回步驟 3](#step-3)

### <span id="error-4">錯誤 4：`openclaw dashboard` 冇打開</span>

先回到步驟 3 再檢查一次 gateway 狀態。如果 gateway 未起好，dashboard 多數都唔會正常打開。

[返回步驟 4](#step-4)

### <span id="error-5">錯誤 5：`openclaw doctor` 出現阻擋性錯誤</span>

先看錯誤訊息指向的是安裝、設定，還是 gateway 問題。大多數情況下，回到步驟 2 或步驟 3 再做一次，就可以排到根源。

[返回步驟 5](#step-5)
