## 先看結論

如果你已經跟住安裝頁裝好 OpenClaw，呢一頁會幫你做 5 個最實用的檢查：確認 CLI 仲用得、看 gateway 狀態、打開 dashboard、跑一次自檢，以及有需要時重跑首次設定。

`gateway` 即是 OpenClaw 在你電腦上的入口服務；`dashboard` 即是本機管理畫面；`doctor` 即是自動檢查常見設定問題的工具。

<div class="loom-note">
  <strong>最短檢查路徑</strong>
  <p>先跑 `openclaw --version`，再用 `openclaw gateway status` 看 gateway 有冇正常回應，之後打開 `openclaw dashboard`，最後用 `openclaw doctor` 做一次自檢。</p>
</div>

## 步驟

### <span id="step-1">1. 確認 CLI 仲用得</span>

先確認 `openclaw` 指令本身仲可以在 Terminal 正常執行。

```bash
openclaw --version
```

做完之後，你應該見到版本號，而唔係 `command not found`。

[睇唔到版本號？看錯誤 1](#error-1)

### <span id="step-2">2. 看 gateway 狀態</span>

再看一次 gateway 狀態。你可以把 gateway 理解成 OpenClaw 在你電腦上的入口服務。

```bash
openclaw gateway status
```

做完之後，你應該見到狀態摘要，而唔係卡住、連線失敗，或者完全冇回應。

[gateway 狀態顯示有問題？看錯誤 2](#error-2)

### <span id="step-3">3. 打開 dashboard</span>

dashboard 即是本機管理畫面。當狀態摘要正常之後，再打開 dashboard 看畫面有冇起到。

```bash
openclaw dashboard
```

做完之後，你應該見到瀏覽器打開本機畫面，或者命令列至少印出一個可以打開的本機網址。

[dashboard 冇打開？看錯誤 3](#error-3)

### <span id="step-4">4. 跑自檢</span>

`doctor` 會幫你檢查常見設定問題。呢一步最適合用來確認你唔係「畫面勉強打開，但其實後面仲有阻擋性錯誤」。

```bash
openclaw doctor
```

做完之後，你應該見到檢查結果；如果有錯，重點睇係咪出現阻擋你繼續使用的致命問題。

[自檢出現阻擋性錯誤？看錯誤 4](#error-4)

### <span id="step-5">5. 需要時重跑首次設定</span>

如果上面任何一步都過唔到，最直接的補救通常係再跑一次首次設定精靈。它會重新帶你走一次基本設定。

```bash
openclaw onboard --install-daemon
```

做完之後，你應該回到可繼續輸入命令的 Terminal，之後再返回步驟 2 到步驟 4 重試。

[首次設定重跑時卡住？看錯誤 5](#error-5)

## 驗證方式

你完成上面 5 步之後，只要下面幾件事都成立，就代表 OpenClaw 大致已經正常啟動：

- `openclaw --version` 有正常回傳版本號。
- `openclaw gateway status` 有回到狀態摘要。
- `openclaw dashboard` 有打開本機畫面，或者印出可打開的網址。
- `openclaw doctor` 沒有出現阻擋你繼續使用的致命錯誤。
- 如果你需要重跑首次設定，重跑之後再做一次步驟 2 到步驟 4，結果應該比之前穩定。

如果其中一項未過，直接跳去下面 `常見錯誤` 的對應項目，再按連結返回相關步驟。

## 常見錯誤

### <span id="error-1">錯誤 1：`openclaw --version` 冇回應或顯示 `command not found`</span>

先重新開一個新的 Terminal 視窗，再試一次。如果仍然唔得，就回到安裝頁重新做 CLI 安裝。

[返回步驟 1](#step-1)

### <span id="error-2">錯誤 2：`openclaw gateway status` 顯示連線失敗、卡住，或者冇狀態摘要</span>

這通常代表 OpenClaw 的本機服務未完全起好。先等一陣再試一次；如果仍然唔得，就去步驟 5 重跑首次設定。

[返回步驟 2](#step-2)

### <span id="error-3">錯誤 3：`openclaw dashboard` 冇打開畫面</span>

先回到步驟 2，再確認狀態摘要是否正常。如果狀態摘要本身已經有問題，dashboard 多數都唔會正常打開。

[返回步驟 3](#step-3)

### <span id="error-4">錯誤 4：`openclaw doctor` 顯示阻擋性錯誤</span>

先看錯誤內容指向的是設定問題、登入問題，還是本機服務問題。最常見的處理方式係回到步驟 5，重跑一次首次設定。

[返回步驟 4](#step-4)

### <span id="error-5">錯誤 5：重跑首次設定時處理唔到舊設定或中途卡住</span>

如果首次設定精靈要求你處理舊設定，通常代表你機器上已經有舊的 OpenClaw 設定。先按畫面提示決定保留、修改或重設；之後再回到步驟 2 繼續檢查。

[返回步驟 5](#step-5)

如果上面都搵唔到你而家遇到的錯誤，可以直接複製錯誤文字，或者截圖目前畫面，然後問 AI 幫你解讀下一步。你只要講清楚自己做到邊一步、見到咩訊息，通常會快好多搵到下一個可試的方向。

## 下一步

- 如果 `gateway status`、`dashboard` 同 `doctor` 都正常，下一步就可以直接開始使用 OpenClaw。
- 如果你發現自己其實未完成基本安裝，先回到 [如何在 Mac 安裝 OpenClaw](/zh-hk/guides/openclaw/mac-install/) 補完安裝主線。
- 如果你仲卡住，先到上面的 `常見錯誤`，再按返回連結逐步重試。
