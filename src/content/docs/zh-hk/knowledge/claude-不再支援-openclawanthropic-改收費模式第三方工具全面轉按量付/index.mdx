# Claude 不再支援 OpenClaw！Anthropic 改收費模式，第三方工具全面轉按量付

## 一句話版本

Anthropic 已由 2026 年 4 月 4 日中午 12 點（太平洋時間）起，停止用戶用 Claude 訂閱額度喺 OpenClaw 呢類第三方工具上跑自動化工作，之後要另外買用量包或直接用 API key，重點係我哋依賴「訂閱覆蓋成本」呢個假設已經失效。

## 點解重要

- **商業模式變咗**
- Anthropic 將第三方工具流量由訂閱制切去按量付費，代表 OpenClaw 呢類產品唔可以再假設「用戶本身有 Claude Pro 就夠」，成本、定價、開通流程都要重新設計。
- **供應商明確區分官方產品同第三方場景**
- 佢哋講到原本訂閱唔係為大量自動化使用而設，仲會優先保障官方產品同 API 客戶，意思係第三方 agent 類工作流喺資源緊張時，優先級本身就較低。
- **高頻自動化被視為壓力來源**
- 報導直接點名 OpenClaw 呢類會連續代做電郵、行程、報到等任務，正正就係最容易放大 token 與算力消耗嘅模式；對我哋嚟講，呢唔係單次政策，而係供應商開始針對 agent 用法分類管理。
- **登入 Claude 帳號唔再等於有可用額度**
- 即使用戶照樣可以用 Claude 登入第三方工具，實際上都要額外買 usage bundles 或提供 API key，這會直接增加首次成功跑通任務嘅摩擦。
- **付款體驗會變成產品能力一部分**
- 既然費用與原本訂閱分開計，用戶之後會問「我今次任務燒咗幾多」「點樣封頂」「用量包幾時用完」，如果產品答唔清，信任感會好低。
- **退款與補償說明供應商預期會有反彈**
- Anthropic 提供一次性補償額度同退款入口，反映今次唔係微調，而係足以影響現有工作流嘅政策轉向；即係話用戶情緒管理同遷移指引會成為實際營運問題。
- **平台風險已經由技術風險升級做業務風險**
- 就算產品功能冇壞，只要上游改計費或權限政策，整條第三方 agent 鏈路都可以突然變得不可用或者不可負擔，呢種風險唔可以再當例外處理。
- **Anthropic 可能正用政策推動生態集中**
- 報導提到市場關注佢哋係咪想引導用戶回流自家工具；無論動機係算力壓力定平台策略，結果都一樣：第三方整合商嘅議價力更弱。
- **同類供應商未必會長期容忍「代操作型 agent」**
- 今次事件提供咗一個訊號：只要某類使用模式特別吃資源、又唔喺供應商最想保護嘅產品面，之後被另立收費甚至限制都好合理。
- **OpenClaw 被視為代表性案例**
- 文章將 OpenClaw 描述成今年初爆紅嘅 AI agent 代表，代表我哋唔係邊緣情境；相反，我哋就係供應商會優先觀察、管制同重定價嘅那一種產品。

## 我哋點睇

- 我哋要即刻停止以「Claude 訂閱可直接驅動 OpenClaw」做預設對外說法，改成清楚講明第三方使用需要獨立額度或 API key，否則用戶一上手就會撞牆。
- 成本控制要前移到控制平面，而唔係留畀模型供應商事後收費；具體要有模型路由、任務預算、封頂、降級策略，同埋可見嘅用量回報。
- 供應商抽換要當成常態設計，不應把任何單一模型商嘅登入方式、補貼政策或訂閱權益寫死入產品核心流程。
- 對外文件要用易明語言交代「有咩變咗、點解你而家可能要另外付費、你會喺邊度見到影響」，否則支援成本會即刻上升。
- 如果我哋繼續做高頻自動化場景，產品定位就唔可以只講能力，仲要證明任務價值高過實際 token 成本，先撐得住。
