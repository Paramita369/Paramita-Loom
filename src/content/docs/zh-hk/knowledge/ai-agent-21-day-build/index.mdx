# 21 天，我用 Claude Code 打造了一整套 AI 分身助理記錄

## 這是什麼內容

- 文章以 21 天實作記錄為主線，描述作者沒有直接採用現成 AI agent 框架，而是先從「可長期延續的記憶系統」著手，解決跨 session、跨裝置失憶問題，逐步打造更貼身的個人 AI 分身。
- 前半段重點在架構設計：用 iCloud、多層記憶、知識圖譜與每日日誌，讓 AI 不必硬記所有內容，而是能穩定找到脈絡、偏好與專案背景。
- 中後段轉向工作流落地與優化，將 Gmail、家庭與財務等流程接入，同時以規則和人工把關高風險動作，並透過週回顧、精簡核心指令、模組化 Skills 與 QA 流程，把 AI 從被動工具推向較主動的協作夥伴。
- 最後作者把重點拉回人本身：AI 真正的價值不在工具新奇，而在使用者是否清楚需求、能判斷品質，並願意承受前期效率下降來換取長期可持續的系統能力。

- 高風險自動化更適合採「AI 生成建議 + 規則或人工決策」模式，而不是端到端全自動。
- 導入 AI 工作流初期出現效率倒退並不罕見；若目標是可持續系統，磨合成本本身就是必要投資。
